## פתיחה
בחלק הקודם של המכינה בנושא הפרונט למדנו על JS וHTML.
אם אתם מכירים HTML ו-Vanilla JavaScript, אז ריאקט (React) היא ספריית JavaScript שנועדה לבנות ממשקי משתמש דינמיים וריאקטיביים. הנה הסבר פשוט כדי להתחיל להבין איך זה עובד:

## מצגת

### מה זה ריאקט?

React.js היא ספריית JavaScript שפותחה על ידי Meta (פייסבוק) לבניית ממשקי משתמש מבוססי קומפוננטות.

במקום לכתוב קוד ארוך עם `document.createElement` או `innerHTML`, ריאקט מאפשרת לך לבנות רכיבים (Components) שניתן למחזר ולנהל בקלות.

## למה לא לכתוב HTML בעצמנו?

אז בעצם לכתוב את כל ה-HTML של אתר בעצמנו זו משימה מאד קשה וסיזיפית וטכנית. בואו ניקח דוגמה קטנה:

```html
<style>
    .item {
        width: 200px;
        height: 100px;
        background-color: aquamarine;
        border-radius: 8px;
        margin: 20px;
        text-align: center;
    }
</style>
<div class="item"> חולצה </div>
<div class="item"> גרביים </div>
<div class="item"> מכנסיים</div>
```

אם ה-HTML כאן הוא קטן אז אין לי בעייה להעתיק קופי פייסט כל פריט בחנות. אבל: אם יש לנו 1000 פריטים בחנות, ואם כל פריט הוא לא רק DIV אחת, אלא יהיה לו שם כפתורים, ו-JS וכל מני דברים נוספים, אני לא יכול לעשות כל כך הרבה קופי פייסט, ואם אני רוצה לעשות שינוי אני צריך לזכור את כל המקומות וכו'.

לכן מה שעושים ברוב המקומות, זה:

**המפתחים לא כותבים HTML בעצמם. הם כותבים קוד שכותב את ה-HTML.**

כאן ב-HTML הכוונה גם ל-CSS ו-JS.

זה נשמע קצת מסובך, וזה אכן קצת מסובך. כי בעצם אני צריך לחשוב על זה מה אני רוצה לכתוב כדי שיצא לי HTML והתנהגות כמו שאני רוצה.

יש לא מעט ספריות ופלטפורמות שעוזרות לעשות את זה.

אנחנו נתעסק עם פלטפורמה מרכזית שהיא היום הכי פופולרית בעיקר לפרוייקטים חדשים ומורכבים, שנקראת ריאקט.

## אתגרים ואופי הלמידה

בניגוד למחזור הקודם שהתעסק בפרונט, שבו למדנו מה שנקרא "מהברזלים" - למדנו ממש איך לכתוב CSS HTML ו-JS ביידים, ממש בצורה שהדפדפן קורא אותם, כאן נעבוד בשיטה קצת שונה.

ביננו לבין התוצאה הסופית בדפדפן יהיו כלים שונים, שעוזרים לנו לייצר ממשקים בקלות רבה יותר.

### היתרונות:
- הם יעשו לנו את החיים יותר קלים ופחות סיזיפיים
- הם בתקווה יתנו לנו פרקטיקות יותר טובות של פיתוח תוכנה (למשל, כמו הדוגמה למעלה עם הקופי פייסט - זה פרקטיקה פחות טובה לעשות קופי פייסט בעצמנו, ופרקטיקה יותר טובה לעשות REUSE)

### החסרונות:
- צריך ללמוד את הכלים עצמם
- בתוך תהליך הלמידה, יהיה הרבה "קסם" (ועל זה נרחיב)

### להבין הכל לעומק מול לדעת להפעיל את הספרייה

יש לנו בהרבה מפגש עם הטכנולוגיה בחיים שלנו את המורכבות הזו, בכל הרמות בחיים:
- האם אני צריך לדעת איך האוטו עובד כדי ליסוע בו?
- האם אני צריך להבין מה התרופה עושה כדי לקחת אותה?

וכו'.

אז כמו בדילמות האלו, גם בתוכנה יש את אותן דילמות. בואו נדבר על הקצוות:

- **קצה אחד - לדעת "הכל"**: אני יודע ומבין כל מה שאני עושה ממש ממש לעומק ולא מתקדם מטר עד שאני מבין מה כל כלי שיש לי ביד עושה
- **קצה שני לדעת רק "איך להפעיל"**: אני לא מבין בכלל איך הכלים שיש לי עובדים אבל אני יודע איך להפעיל אותם.

במחזור הזה, ננסה להביא איזון בין שניהם.

כמה נקודות לגבי זה:

- צריך לומר שגם לכל אדם יכולים להתאים דברים אחרים בסיטואציות שונות.
- לפעמים אני מרגישה שאני רצה בעשייה בלי להבין למה עושה ככה, ולפעמים הפוך: שמדברים הרבה "תיאוריה" ולא מבינים איך זה קשור למשימות שלנו.
- ננסה להיות בתקשורת על זה, ולהגיד כשזה ככה.
- בהתחלה ובחלקים מסויימים נהיה יותר ב"לעשות כמו תוכי"\ נעשה ונשמע כי זה יותר יתאים
- יהיו חלקים שיהיו יותר חפירות אם לדעתי יהיה חשוב להבין את הרקע מאחורי הדברים.

ספציפית בקורס הזה גם נלמד הרבה דברים שקשורים ל-JS ולפיצ'רים "חדשים" שקשורים ל-JS שמתמשים בהם הרבה בריאקט.

## התקנות כלים

נעבוד על הריפו [https://github.com/drorsoft/react-course](https://github.com/drorsoft/react-course) שה-main שלה אמור להיות מעודכן תמיד לאיפה שאנחנו.

אז אנחנו נתחיל עם 2 כלים בסיסיים שנכיר. כבר עבדנו בעבר עם VS CODE והדפדפן.

### כמה מילים על NODE.JS:

במחזור הקודם דיברנו על JS, שזו שפה שנוצרה לשם הרצת סקריפטים בצד הקליינט. יש לה כל מני יתרונות וחסרונות שבהם גם ניגע לעומק במחזור.
בשנת 2009 משהו בשם ראיין דאל השתמש בJS כדי ליצור סביבת שרת שכתובה ב-JS. הוא קרא לה NODE JS.

עכשיו NODE JS תוכננה בתור שרת אינטרנט, שמבקשים ממנו בקשות והוא מחזיר תשובות וכו'. ועדיין משתמשים בו לשם כך, למשל, אחד השרתים במערכת ההרשמה הוא שרת של NODE JS שמריץ קוד של JS.

אבל, היום הוא גם כלי מאד חשוב שמשמש בתור שרת פיתוח.

#### מה זה שרת פיתוח?

זה בעצם תוכנה קטנה, שהיא למעשה שרת אינטרנט לכל דבר ועניין, שכל התפקיד שלה זה לעזור לכם לפתח את היישום שלכם.

בנוסף, יש כלי נוסף שנקרא NPM. NPM בא עם נוד, למרות שגם אפשר להתקין כל אחד בנפרד.

NPM זה **NODE PACKAGE MANAGER**

מה זה אומר?
זה כלי שעוזר לנו לנהל חבילות. מהם חבילות? אנחנו נעבוד עם הרבה חבילות בהמשך, אבל "חבילה" שלפעמים קוראים לה גם ספרייה או framework תלוי בהקשר - היא בעצם אוסף של הרבה קוד, שעוזר לנו לעשות דברים. למשל ריאקט היא חבילה שמכילה הרבה קבצי JS.

אז אנחנו נתקין את נוד, ואז נתקין ריאקט, ואז נקרה מה בעצם קורה:

### התקנת NODE

1. נכנסים ל [https://nodejs.org/en/download](https://nodejs.org/en/download)
2. מורידים את גרסא 22.

ואז כדי לראות שזה עובד לנו, אנחנו לוחצים בטרמינל:

```bash
node -v
```

ואז אנחנו אמורים לראות את הגרסא.

למשתמשי וינדוס צריך לוודא שהקובץ נכלל ב-PATH.

אחרי זה אנחנו נתחיל להוריד את ריאקט. אז נכון שאמרתי לכם שיהיו הרבה כלים?

אז אנחנו לא נוריד ישירות את ריאקט, אלא נוריד את vite. נרחיב על זה בהמשך אבל VITE זה כלי לפיתח ובנייה של ישומי אינרנט, והוא בעצם יעזור לנו (יתן לנו פיגומים להשתמש בריאקט לפיתוח, וגם לפרודקשין, זאת אומרת לפרוס \ לפרסם את הריאקט שלנו בתור אתר)

אז אנחנו מקלידים:

```bash
npm create vite@latest
```

ואז עונים על השאלות:
- הוא יבקש מכם להתקין את VITE, לענות Y
- שם הפרוייקט מה שאתם רוצים כרגע תקראו לזה REACT PROJECT
- בחירת פריימוורק - לבחור ריאקט
- ווריאנט - תבחרו JAVASCRIPT

עכשיו נוצרו לנו כמה קבצים. אבל עדיין לא גמרנו.

צריך להכנס לתקייה של הפרוייקט:
```bash
cd react-project
```

ואז להריץ:
```bash
npm install
```

עכשיו אנחנו לא נעבור על כל הדברים לעומק, אבל אני רוצה כן לעבור בקלילות על מה יש לנו בעצם עכשיו בפרויקט ובפעמים הבאות נתחייס לרוב הדברים:

- קודם כל יש לנו קובץ שנקרא `README` שנותן לנו מידע בססי על הפרוייקט
- יש לנו תקייה שנקראת `NODE_MODULES` זו תקייה שלא נוגעים בה, היא תקייה מאד גדולה שמכילה את כל ההתקנות שהתקנו.
- בנוסף יש לנו קובץ HTML, שאנחנו מכירים.
- עכשיו, יש לנו קובץ שנקרא `PACKAGE.json`.

הקובץ הזה בעצם מרכז לנו הרבה מידע על הפרוייקט, נעמיק בו בהמשך, אבל מה שהכי חשוב לנו זה החלק של ה-scripts.
אלו הפקודות שאנחנו נשתמש בהם כדי לעבוד על הפרוייקט. תריצו את הפקודה:

```bash
npm run dev
```

- ואנחנו צריכים חנות שתתקין לנו אפליקציות, נכון? אז זה כמו NPM
- ואנחנו לא יכולים פשוט להתקין וורד נכון? צריך להתקין TEAMS שזו חבילה של ישומים. ורק אז בסוף הגענו למה שאנחנו רוצים ויכולים לפתוח WORD. אז ריאקט זה דומה לכך.

עכשיו מכיוון שחפרנו הרבה, ננסה לעבור טיפה על הקוד של ריאקט, ולראות מה קורה שם, ובעיקר ננסה לשנות אותו קצת.

אז קודם כל יש לנו קובץ עם סיומת מוזרה, שזה JSX. מזה JSX? זה בעצם שעטנז מוזר בין HTML ל-JS. נעמיק בזה בהמשך, אבל בעצם, כל קומפוננטת ריאקט, היא בעצם פונקציה, שמחזירה, JSX שזה סוג של HTML.
